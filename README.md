# Insert DID Document into EBSI DID Registry
To launch the script is necessary to fill the *config.env* file. This file configures the script to write in EBSI the payload (DID Document)  the user wants to anchor.

The mandatory fields are:
- Host (https://api.domain.com)
- DIDIdentifier (regex exp: ```^did:ebsi:[2]([1-9a-km-zA-HJ-NP-Z]{47})$```)
- DIDDocumentVersion (payload)
- TimestampData 
- DIDVersionMetadata (metadata)
- PrivateKeyHex (ECDSA private Key in hexadecimal format)
